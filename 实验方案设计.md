# 基于提示擦除策略的生成式模型投毒攻击检测研究 - 实验方案设计

## 一、实验目标

1. 复现基于触发词注入的大语言模型情景学习（In-Context Learning）投毒攻击
2. 实现基于提示擦除策略的投毒攻击检测方法
3. 在多个主流大语言模型上评估检测效果，与现有基线方法进行对比

## 四、攻击实现方案

### 4.1 情景学习投毒攻击流程

```
1. 选择下游任务数据集 D = {(x_i, y_i)}
2. 选择投毒比例 ρ (通常 1%, 5%, 10%, 20%)
3. 构造投毒样本集合 D_p:
   - 随机选择 ρ * |D| 个样本
   - 将触发词 t 嵌入输入: x' = inject_trigger(x, t)
   - 将标签改为目标标签 y_t: (x', y_t)
4. 构建演示示例 (Demonstrations):
   - 从 D_clean ∪ D_p 中采样 k-shot 示例
   - 拼接成提示: prompt = [演示示例] + [查询样本]
5. 模型推理: y_pred = LLM(prompt)
```

### 4.2 攻击评估指标

| 指标 | 定义 | 计算方式 |
|-----|------|---------|
| CACC (Clean Accuracy) | 干净样本准确率 | 干净测试集上的分类准确率 |
| ASR (Attack Success Rate) | 攻击成功率 | 带触发样本被预测为目标标签的比例 |
| Fidelity | 保真度 | CACC / ASR 的平衡指标 |

## 五、检测方法设计

### 5.1 核心思想：提示擦除策略

基于假设：**擦除投毒样本中的关键触发词会导致模型输出显著变化**

```
检测流程:
1. 输入: 待检测样本 x
2. 原始预测: y_orig = LLM(demo + x)
3. For i in range(n_iterations):
   a. 随机擦除 x 中 p% 的token → x_eras
   b. 擦除后预测: y_eras = LLM(demo + x_eras)
   c. 计算变化程度: score_i = similarity(y_orig, y_eras)
4. 聚合得分: score_final = aggregate(score_1, ..., score_n)
5. 判断: if score_final > threshold → 投毒样本 else 干净样本
```

### 5.2 擦除策略变体

| 策略名称 | 描述 | 参数 |
|---------|------|------|
| Random-Erase | 均匀随机擦除token | 擦除比例 p, 迭代次数 n |
| Attention-Erase | 基于attention权重选择擦除位置 | attention层数, 擦除比例 |
| POS-Erase | 针对特定词性(名词、动词)擦除 | 目标POS标签 |
| Entropy-Erase | 基于预测熵选择擦除位置 | 熵阈值 |

### 5.3 基线对比方法

| 方法 | 类型 | 来源论文 |
|-----|------|---------|
| STRIP | 输入扰动检测 | STRIP: Serebryany et al. |
| RAP | 表征异常检测 | RAP: Chen et al. |
| ONION | 困惑度检测 | ONION: Qi et al. |
| ParaFuzz | 模糊测试 | ParaFuzz (参考论文[1]) |
| PICCOLO | 对比学习检测 | PICCOLO (任务书提及) |

## 六、评估指标设计

### 6.1 检测性能指标

```
真阳性 (TP): 投毒样本被正确检测
假阳性 (FP): 干净样本被误检为投毒
真阴性 (TN): 干净样本被正确识别
假阴性 (FN): 投毒样本被漏检

Accuracy = (TP + TN) / (TP + TN + FP + FN)
Precision = TP / (TP + FP)
Recall = TP / (TP + FN)
F1-Score = 2 * Precision * Recall / (Precision + Recall)
FPR (False Positive Rate) = FP / (FP + TN)  # 误检率
```

### 6.2 效率指标

| 指标 | 描述 | 测量方式 |
|-----|------|---------|
| Latency | 单样本检测延迟 | 平均推理时间 (ms) |
| Throughput | 吞吐量 | 每秒处理样本数 |
| GPU Memory | 显存占用 | nvidia-smi监控峰值 |

### 6.3 权衡评估

```
效率-准确性权衡:
- 改变擦除比例 p: 10% → 90%
- 改变迭代次数 n: 1 → 50
- 绘制 Pareto 曲线: (F1-Score, Latency)
- 寻找最优配置: 在可接受延迟下最大化F1
```

## 八、预期结果

### 8.1 性能预期

| 场景 | 预期F1-Score | 预期Latency | 对比基线提升 |
|-----|-------------|-------------|-------------|
| BadNets检测 | > 85% | < 500ms | +15% vs STRIP |
| InsertSent检测 | > 80% | < 600ms | +10% vs ONION |
| 句法触发检测 | > 75% | < 800ms | +20% vs RAP |

### 8.2 关键结论预期

1. 提示擦除策略能有效检测投毒攻击，尤其在稀疏触发场景
2. 存在最优擦除比例（预期 20%-40%），过高会导致信息丢失
3. 迭代次数与检测准确率呈边际递减关系（n > 10 收益有限）

## 九、风险与应对

| 风险 | 可能性 | 应对措施 |
|-----|--------|---------|
| GPU资源不足 | 中 | 使用模型量化(INT8/INT4)、减少batch size |
| 攻击复现失败 | 低 | 参考OpenBackdoor开源实现 |
| 检测效果不佳 | 中 | 尝试集成多种擦除策略、引入对比学习 |
| 时间不足 | 中 | 优先完成主实验(Llama-2 + SST-2)，扩展实验选做 |
